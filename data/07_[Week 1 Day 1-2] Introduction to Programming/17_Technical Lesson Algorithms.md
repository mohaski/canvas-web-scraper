**Estimated Completion Time:** 30 min.

In this lesson, we will explore how to apply the SDLC approach to plan and solve a Python coding problem. Specifically, we will focus on creating a program that calculates the factorial of a given number. By following the SDLC process and utilizing pseudocode, you will learn how to effectively plan, develop, and maintain Python programs in a structured and organized manner.

**Overview of the SDLC Phases**

As you have learned, the SDLC is a systematic process used by software developers to design, develop, and test high-quality software. It ensures that the final product meets the user’s requirements and is free of bugs. The typical phases of the SDLC include:

* Planning and requirements gathering
* Design
* Implementation (coding)
* Testing
* Deployment
* Maintenance

*​*

Remember, this is a simplified example. In real-world scenarios, the SDLC process can be more extensive and may involve additional steps and considerations depending on the project complexity and requirements.

### Instructions

Review each phase more below.

### [Phase 1: Planning and Requirements Gathering](#dpPanel0)

* **Objective:** Understand the problem and gather necessary requirements.
* **Problem Statement:** We need to create a program that calculates the factorial of a non-negative integer.
* **Inputs:** The program should accept a non-negative integer as input.
* **Outputs:** The program should return the factorial of the given number.
* **Edge Cases:** Consider invalid inputs, such as negative numbers or non-integer values, and how to handle them gracefully.

**Action Steps:**

* Write down the problem statement clearly.
* Identify what the input and output should be.
* Think about any unusual cases and how the program should handle them.

### [Phase 2: Design and Analysis](#dpPanel1)

**Objective:**Plan the solution and outline the algorithm using pseudocode.

1. **Break Down the Problem:**

* **Validate Input:** Ensure the input is a non-negative integer.
* **Computer Factorial:** Use a loop or recursive function to calculate the factorial.
* **Display the Result**: Show the calculated factorial to the user.

1. **Pseudocode Outline**: Writing pseudocode helps to structure your thoughts and design the solution before actual coding.

**Example Pseudocode:**

```
function calculate_factorial(number):  
    if number is less than 0 or not an integer:  
        display an error message  
        return  
      
    factorial = 1  
    for each value i from 1 to number:  
        factorial = factorial * i  
      
    return factorial  
  
main:  
    prompt the user to enter a number  
    call the calculate_factorial function with the user input  
    display the calculated factorial
```

**Action Steps:**

* Create a clear step-by-step pseudocode outline.
* Ensure your pseudocode covers all aspects of the problem, including edge cases.

### [Phase 3: Implementation](#dpPanel2)

**Objective:** Convert the pseudocode into functional Python code.

**Example Python Code:**

```
def calculate_factorial(number):  
    if number < 0 or not isinstance(number, int):  
        print("Error: Invalid input. Please enter a non-negative integer.")  
        return None  
      
    factorial = 1  
    for i in range(1, number + 1):  
        factorial *= i  
      
    return factorial  
  
# Main program  
user_input = input("Enter a number: ")  
try:  
    number = int(user_input)  
    result = calculate_factorial(number)  
    if result is not None:  
        print("The factorial of", number, "is:", result)  
except ValueError:  
    print("Error: Invalid input. Please enter a valid integer.")
```

**Action Steps:**

* Translate the pseudocode into Python, ensuring each step is accurately implemented.
* Pay attention to details, such as input validation and error handling.

### [Phase 4: Testing](#dpPanel3)

**Objective:** Verify the program works as expected with various test cases.

1. **Prepare Test Cases:**

* **Valid Inputs:** Test with numbers like 0, 1, 5, 10.
* **Invalid Inputs:** Test with negative numbers, non-integer values (e.g., -5, 3.14) and non-numeric strings (e.g., "abc").

1. **Execute Tests and Verify Results:**

* Run your program with the prepared test cases.
* Ensure the outputs are correct and handle errors gracefully.

**Action Steps:**

* Develop a set of test cases covering typical and edge scenarios.
* Execute the tests and observe if the program behaves as expected.
* Debug and fix any issues that arise during testing.

### [Phase 5: Deployment](#dpPanel4)

**Objective:** Deploy the program for users to access and use.

* **Prepare for Deployment:**
  + Ensure the code is ready and works as expected in the intended environment.
  + Document the steps for running the program.
* **Deployment Steps:**
  1. Deploy the code to the target environment (e.g., a web server, a command-line tool).
  2. Provide clear instructions for users on how to run and use the program.

**Action Steps:**

* Finalize the program and prepare it for deployment.
* Write documentation and user guides as needed.

### [Phase 6: Maintenance](#dpPanel5)

**Objective:** Keep the program up-to-date and address any issues post-deployment.

* **Monitor Performance:**
  1. Observe how the program performs in real-world use.
  2. Gather feedback from users.
* **Update and Maintain:**
  + Address any bugs or issues reported by users.
  + Make updates or improvements based on feedback and evolving requirements.

**Action Steps:**

* Set up a system to collect and review user feedback.
* Plan regular updates and maintenance to keep the program functional and efficient.
