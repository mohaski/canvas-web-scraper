**Prompt Engineering** is the skill of crafting effective instructions for AI models to get desired outputs. In a workplace setting, this has become increasingly important as teams integrate AI tools into their workflows. So much so that the title ‘Prompt Engineer’ has begun to show up in job postings and descriptions.

### Prompts

---

Instructions given to AI models

### Context

---

Background information provided

### Constraints

---

Limitations or specific requirements

Proper prompt engineering leads to more precise, useful outputs from Generative AI. In daily work, this skill helps:

* Reduce back-and-forth with AI tools
* Get more accurate results
* Save time on routine tasks
* Maintain consistency in outputs
* Better document requirements

#### **Example**

Imagine you're a junior developer tasked with creating an API endpoint. Consider the differences between the two prompts below and how the second one utilizes context and constraints resulting in a better prompt and hopefully more in depth response.

* **Initial prompt:** "Create a REST API endpoint for user registration"
  + Too basic and broad, does not specify coding language or tool, does not provide details on how you want it to function.
* **Engineered prompt:** "Create a Python Flask REST API endpoint for user registration that: Accepts email and password, validates email format, hashes password, returns a JSON response, and includes error handling"
  + Specifies context (flask and python) and provides constraints (what it can do).
