**Scenario:** You are working as a data analyst for a small start up company that is attempting to break into the phone app market and wants to know some general trends. As of right now, they plan to initially target the android/google play store market. You have been tasked with analyzing and displaying (dashboard) general trends in app success across the google play store.

Generative AI can dramatically accelerate data analysis code development by providing scaffolding for common analytical patterns, suggesting optimizations, and helping debug issues. Throughout this lab assessment you are expected to attempt to utilize the help of a generative AI tool via good prompt engineering practices and techniques. Consider all areas of the project, from initial question setting all the way through to design layout for your dashboards.

Developers should carefully verify the generated code's correctness, test edge cases, and maintain good software engineering practices rather than blindly implementing AI suggestions. The technology works best as an intelligent assistant that augments human expertise rather than replaces it, helping analysts focus more on insights and interpretation while reducing time spent on boilerplate code and common implementation patterns.

This lab consists of two complementary parts that will evaluate your understanding, application, and critical thinking around appropriate visualizations for dashboards. As a reminder, here's an overview of each part of the lab, utilizing the CRISP-DM process you will be following.

### Part 1: Data Analysis and Dashboard Creation (80 Points)

**Step 1: Download and load in Data**

* Utilize Pyspark syntax, should avoid base Pandas at this point.

**Step 2: Business Understanding**

* Formulate analytical questions to answer based on the scenario and business problem provided. These questions should drive your subsequent steps and analysis below. There is no ‘right’ answer here. Start simple and increase complexity if time is available.

**Step 3: Data Understanding (Initial EDA)**

* Explore the dataset provided utilizing Pyspark to perform basic EDA.
  + Consider things like counts, unique values, null values, mismatched datatypes, value distributions, and any ‘messy’ data that might need to be cleaned.
* Formulate any necessary data preparation/cleaning tasks for step 4.

**Step 4: Data Preparation (Cleaning)**

* Prepare and clean the data as necessary to accurately address the questions and business problems specified above.
  + Utilize a combination of Pyspark and Pandas code when needed.
* Consider implementing data validation as a part of this process.
* End Result: Save cleaned data file as new csv for use in Tableau.

**Step 5: Data Analysis**

* Utilize the cleaned dataset from step 4 within Tableau to explore analytical questions and provide data driven recommendations via dashboard display.
* Initial exploration and visualization could occur in notebook as well.
* Produce at least 6 visualizations that address the business context and questions.

**Step 6: Dashboard Creation**

* Design and develop a multi-page dashboard that incorporates user interactivity in order to address the business problem.
* Make sure to include parameter controls, filtering, and proper navigation buttons.

**Step 7: Documentation**

* Create comprehensive technical documentation

### Part 2: Reflection (20 Points)

**Step 8: Reflection**

* As a part of the required submission you need to record a canvas video discussing your thought process and selection of appropriate visualizations for the dashboard.
* Highlight any challenges with the data involved in the cleaning process.
* Discuss any relevant decisions in dashboard design or data preparation that affected your analysis or direction of analysis.
