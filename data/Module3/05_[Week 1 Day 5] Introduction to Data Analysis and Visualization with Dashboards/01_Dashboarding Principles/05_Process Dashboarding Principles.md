### 1. Define Objectives

* Identify the target audience
  + Will set the tone and scope of the dashboard
* Determine key questions to answer
  + What questions and analytics need to be looked at
  + Create a hierarchy of importance to help with F-pattern

### 2. Gather and Prepare Data

* Collect any necessary data
* Clean and pre-process as necessary
  + Handle null values
  + Handle duplicate data
  + Standardization of strings

### 3. Define Metrics and KPIs

* Select specific metrics to track and visualize
* Should work to address the questions defined in step 1
* Focus on only one or two metrics at a time to prevent information overload

### 4. Select Appropriate Visuals

* Need a visual to address each question
* Choose the best chart type that aligns with the KPI metric

### 5. Design the Layout

* Plan out the structure and location of visuals on your dashboard (outline)
* Consider F-pattern information hierarchy and user flow
* Select contextual color themes using color theory
* Consider non-visualization graphics for thematic value and flair

### 6. Develop the Dashboard

* Connect to cleaned data source
* Create and implement visuals within a dashboarding tool
* Maintain consistent styling, coloring and theme across visuals
* Add Interactivity:
  + Implement user filtering if relevant
  + Allows user to drill-down into more granular level analysis

### 7. Deploy and Maintain

* Launch dashboard to hosting service or via company platform
* Setup process for real-time updates if desired
* Plan for way to receive feedback from user

### 8. Document and Refine

* Update and adapt as needed based on feedback and use
* Document how a user can interact with the dashboard
* Highlight key aspects of the dashboard to pay attention to
