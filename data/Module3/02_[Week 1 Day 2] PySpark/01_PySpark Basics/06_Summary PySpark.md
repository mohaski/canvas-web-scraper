### Terms Recap

* **Apache Spark:** Distributed computing system for handling big data.
* **Cluster:** Multiple computers working together for parallel processing.
* **SparkContext & SparkSession:** Interfaces to Spark’s APIs; SparkContext for unstructured RDDs, SparkSession for structured DataFrames.
* **RDDs & DataFrames:** RDDs manage distributed, unstructured data; DataFrames manage structured data with SQL support.
* **MLlib:** Spark’s machine learning library for scalable model training.

### Key Concepts

PySpark’s distributed processing, data management (RDDs and DataFrames), and MLlib enable scalable data manipulation and machine learning on large datasets, enhancing efficiency and performance in data-intensive tasks.

### Brief Process Overview

1. Initializing SparkSession or SparkContext to connect to the cluster.
2. Loading data into RDDs or DataFrames, depending on its structure.
3. Applying transformations (e.g., filter, map) to structure and prepare data.
4. Executing actions (e.g., count, collect) to trigger processing.
5. Using MLlib for machine learning tasks if needed.
