The amount of data that the average analyst works with is huge, but it is tiny compared to all the data generated by a large organization. In fact, it’s extremely unlikely – almost impossible – that an analyst or data scientist would ever need to work with ALL of an organization’s data.

That’s where the subject of this module comes in: filtering, ordering, limiting and grouping data.   
Instead of reading all of the data into a Pandas DataFrame or an Excel spreadsheet, it’s much more efficient to reduce and structure the data as much as possible using SQL before beginning your analysis.

### Industry Example: Meta/Facebook

![](https://moringa.instructure.com/courses/1406/files/625439/download)
Large organizations produce enormous amounts of data, but it’s almost impossible to imagine a situation where an analyst needs to work with all of it.

Meta/Facebook have offices all over the world, but an analyst with HR probably doesn’t need to look at all of their offices at once. Instead, they can filter out the offices and employees they don’t need to look at.

A data scientist who wants to rank their company’s top ten products by popularity could read all the products into a DataFrame and then order it by sales and sort accordingly. If they know that’s what they want, though, it’s much faster to query the data to only return those results.

### Learning Outcomes

After completing this module, you should be able to:

* Add filters to SQL queries using the WHERE statement, and refine those filters using conditional operators.
* Order the results of your queries by using ORDER BY (ASC & DESC).
* Limit the number of records returned by a query using LIMIT.
* Group data for higher level analysis using GROUP BY.
