### Version Control with Git

Think back to a time when you worked on a group project or a complex task with multiple revisions. How did you manage everyone's contributions? How did you track changes and correct mistakes? You might have encountered a situation similar to the one shown in the image below - a folder filled with numerous versions of the same document, each with a slightly different name. Sound familiar?

![computer folder with confusing file version names](https://moringa.instructure.com/courses/1406/files/624738/download)

While this approach might work for small, personal projects, it quickly becomes unmanageable in the world of data science and software development. We need a more sophisticated, programmatic solution. This is where version control software comes in, and specifically, Git.

Git is a powerful tool that revolutionizes how we manage changes in our projects. It allows us to:

1. Track changes to files over time
2. Collaborate seamlessly with multiple people
3. Maintain a comprehensive history of our project
4. Easily revert to previous versions if needed

As an industry-standard with a vast user base, Git is an essential skill for any aspiring data scientist. It integrates with the data science workflow we discussed earlier, supporting the iterative nature of the CRISP-DM process and facilitating collaboration among different roles in a data science team.

In this lesson, we will walk through the process of installing Git on your system, taking the first concrete step to set up your data science environment.

### How Does Git Work?

Git records snapshots of files and uses those snapshots to trace changes made to files during the life of a project. These snapshots are called ‘commits.’

As files change over time, new commits are created by us, the users. As different developers work on a project, they add their own commits. When it is time to bring all the work back together into one project, git manages this process for us by looking at all the commits, seeing where two versions of a file are different from each other, and combining the differences in a new commit.

### Git Terminology

Git Terminology with definition

| Term | Definition |
| --- | --- |
| Environment | A term to describe the programs installed on a developers or data scientist’s computer. |
| Development Environment | Like “environment” but usually more focused specifically on the programs used to write code; often interchangeable with “environment.” |
| Version Control System | Computer program built to track changes made to files. |
| Git | One of many varieties of version control system; industry standard in software development and data science. |
| GitHub | Online social network used for hosting git-based projects. |
| Git Bash | A command line tool installed on Windows computers when git is installed. |
| Commit | A snapshot of a file taken by git. |
| Repository (Repo) | A computer directory that is being tracked using git. |
| Local | The physical computer you are working on right now; in git terms, the repository on whatever machine you are physically working on right now. |
| Remote | Any computer you have access to via the internet; in git terms, a remote is a repository that is not on your physical machine. |
| Command Line; Command Line Interface (CLI) | A plain text (not graphical) system for interacting with a computer. |
| Graphical User Interface (GUI) | A graphical system for interacting with a computer (e.g., Microsoft Windows or the MacOS Finder). |
