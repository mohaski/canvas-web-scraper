Writing a Subquery includes the following steps:

1. Identify the data you require, your goal in running the query.
2. Break down that goal into smaller parts. Remember that with subqueries we start with the smaller question.
3. Write that smaller query.
4. Use the smaller query as part of the main query’s SELECT, FROM, or WHERE clause.  
   1. In general, if you are trying to create a new column, the subquery goes in the SELECT statement.
   2. If you are trying to filter the data, the subquery goes with the WHERE clause.
   3. If you are trying to create an entirely new table, it goes in the FROM clause.  
      1. This is typically the rarest use-case, but can happen when the overall query simply gets too complicated and you want to keep your code readable.
5. Examine the results and refine future queries.
