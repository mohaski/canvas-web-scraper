For this topic, we took a look at how tabular data is structured by row and column with rows being used to represent observations and columns being used to represent specific features.

We went over how tabular data could be stored in pure text files such as CSVs with new line characters and delimiters:

* Delimiters being characters used to demarcate entries belonging to different columns on a given row
* New line characters (‘\n’) demarcating the end of a row
* With CSVs, the most popular text based tabular format, using comma (‘,’) delimiters

We then learned how to load CSVs into Python data structures and vice-versa using the csv module. Some key objects that we encountered for this are the csv.reader(), csv.DictReader() for reading in CSV and csv.writer() and csv.DictWriter() for writing to CSV.

Once the CSVs were loaded, you used your knowledge of Python data types and data structure wrangling to then perform cleaning and transformation operations and finally save the results to file.
