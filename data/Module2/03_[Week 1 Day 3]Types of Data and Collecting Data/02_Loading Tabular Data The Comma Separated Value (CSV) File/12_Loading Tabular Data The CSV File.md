Tabular data – information organized in rows and columns – is ubiquitous. Most organizations store their data in this form within the various databases they maintain. This all makes sense: it is a logical and transparent way to store information where we collect data on a fixed set of attributes for a set of transactions or objects/persons of interest. In fact, you have already played around with this kind of data in the previous module. 

**CSVs (Comma-Separated Values)** are a common text file format for storing this kind of data, using commas (or other delimiters like tabs or semicolons) to separate column entries for a given row. Data analysts and data scientists encounter CSVs frequently. While organizations often store their tabular data in large relational databases, a particular analysis might only need a small subset of that data. CSVs are these curated excerpts – analysts can request specific data sets from the database and receive them as manageable CSV files for further analysis in Python or other tools. 

In this topic, we introduce the Python csv library and explore the ways it can facilitate the process of reading from and writing to CSVs.  In particular, you will see how the csv library can be used to interpret data types during the loading process and store the processed information in relevant Python data structures: namely, in nested lists or in a list of dictionaries.
