### File Handling

Refers to the techniques used to open, read, write, and close files on your computer's storage. This allows programs to interact with data stored outside of their memory space.

#### How does it work?

Python provides built-in functions like open(), read(), write(), and close() to manage files. You can open a file in various modes (read, write, append) and perform operations based on your needs.

### Logging

Involves recording messages and events that occur during a program's execution. These logs can be used for debugging, monitoring program behavior, or auditing system activity.

#### How does it work?

Python offers the logging module for structured logging. Modules are scripts that contain Python code, especially functions, and allow for the module to be included in subsequent scripts. This module allows you to define different logging levels (debug, info, warning, error, critical) and write messages to various destinations (console, files, network).

### Conceptualize File Handing and Logging

Imagine a program as a chef in a kitchen.

* **File Handling**: Think of ingredients as data stored in the pantry (files). The chef (program) uses tools (functions) to retrieve ingredients (read), prepare them (process), and store leftovers in the refrigerator (write) for later use.
* **Logging**: The chef in the kitchen uses a notepad where the chef (program) jots down notes (messages) about the cooking process (program execution). These notes can be categorized by importance (logging levels) and reviewed later (analysis) to identify any issues or areas for improvement.
